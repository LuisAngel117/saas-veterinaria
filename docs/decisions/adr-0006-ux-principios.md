# ADR-0006 — Principios UX/UI

## Contexto
El objetivo es demo vendible (2–3 min) en Español.

## Decisión
- UI en Español, flujos por rol.
- Error handling claro (mensajes útiles + no bloquear sin razón).
- Flujo core demo: crear cita → atender → cerrar → facturar → pago → reporte.
- Permisos reflejados en UI (ocultar/deshabilitar con explicación).

## Consecuencias
- La UX debe priorizar rapidez y consistencia.
- Seeds deben permitir probar el flujo completo sin configuración manual.
